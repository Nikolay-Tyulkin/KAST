# Security Policy

## Supported Versions

KAST is currently maintained from the main development branch. Security fixes are
handled on a best-effort basis for the current firmware and documentation.

## Reporting a Vulnerability

This is a local-device project without a cloud backend. If you find a security or
safety-relevant issue, open a GitHub issue in this repository and describe the
problem clearly.

Please include:

- the affected firmware version, if known;
- the affected hardware setup;
- steps to reproduce the issue;
- relevant logs or screenshots;
- expected and actual behavior.

Do not include private Wi-Fi credentials, personal data, serial numbers, or other
sensitive information in public logs or screenshots.

## Scope

Security-relevant reports may include:

- unsafe or unexpected device behavior;
- Wi-Fi settings portal exposure issues;
- persistent storage behavior that leaks sensitive settings;
- build or release artifact concerns;
- dependency vulnerabilities that affect the firmware.

General bugs and feature requests should use the matching issue templates.

## Response Expectations

Reports are reviewed on a best-effort basis. If an issue is confirmed, the fix
will be tracked publicly unless sensitive details require limited disclosure.
