# SKILLS

Индекс проектных навыков для `knitting_assistant`.

Важно: opencode загружает навыки из `.opencode/skills/<name>/SKILL.md`, не из этого файла.

## Активные skills
- `product-spec` (`.opencode/skills/product-spec/SKILL.md`) — когда нужно зафиксировать требования устройства: кнопки, ряды, экран, батарея, ограничения платы.
- `implementation-plan` (`.opencode/skills/implementation-plan/SKILL.md`) — когда нужно разложить согласованные требования на этапы для embedded-реализации.
- `quality-gate` (`.opencode/skills/quality-gate/SKILL.md`) — перед финальным ответом по задаче: что проверено, что не проверено, какие риски остались.

## Мини-правила
- Не писать код во время `product-spec` и `implementation-plan`, если пользователь явно не просил переходить к реализации.
- Для UI и логики использовать термин "ряды".
- Перед финальным ответом по реализации использовать `quality-gate`: просмотреть изменения, запустить доступные проверки и явно указать непроверенные области.
- Подтвержденная проверка прошивки: активировать ESP-IDF профиль через `. "C:\Espressif\tools\Microsoft.v6.0.1.PowerShell_profile.ps1"`, затем выполнить `idf.py build`.
- Подтвержденная прошивка платы: `idf.py -p COM14 build flash`, если пользователь не указал другой порт.
- Перед прошивкой, которую выполняет ассистент, увеличить `PROJECT_VER` в корневом `CMakeLists.txt` на следующий `0.0.1-dev.N`; версия должна отображаться внизу экрана после прошивки.
- Для батареи помнить mapping ESP32-S3: Waveshare `GPIO1/BAT_ADC` соответствует ESP-IDF `ADC_CHANNEL_0`; делитель напряжения `200k/100k`, коэффициент `3`.
- Для логов и smoke-проверки устройства использовать `idf.py -p COM14 monitor`; выход из монитора `Ctrl+]`.
- Отдельных тестов/линта сейчас нет; не выдумывать команды без появления соответствующих конфигов.
